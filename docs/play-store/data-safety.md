# Play Data safety form — answers

Play Console → Policy → App content → **Data safety**.

**Does your app collect or share any of the required user data types?** → **No**

Indirimbo Zikundwa:
- has no account, login, analytics, ads, or third-party tracking SDKs;
- stores settings and favorites **on the device only** (not transmitted);
- "Live share" is peer-to-peer on the local Wi-Fi — there is **no server** and
  nothing is sent to the developer;
- the optional content-update check is a plain GET of a public dataset file
  (no user data is sent).

So the form answers are:
- Data collected: **None**
- Data shared: **None**
- (Because nothing is collected, the follow-up questions about encryption in
  transit / data deletion don't apply.)

**Privacy policy:** https://indirimbo-zikundwa.github.io/privacy.html
(states: no personal data collected; settings/favorites stay on device; live
share uses the local network only and runs no server.)
